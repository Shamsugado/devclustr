# Current Feature: Resizable Item Drawer

## Status

Not Started

## Goals

- Add a draggable resize handle on the left edge of the item drawer so users can widen/narrow it horizontally.
- Persist the user's chosen drawer width (e.g. localStorage) so it's remembered across sessions/reloads.
- Constrain resizing to sensible min/max bounds (min ~ current default width, max ~90vw or a fixed cap) so it never overlaps the sidebar or becomes unusably narrow.
- Only enable resizing at `sm` breakpoint and up — mobile drawer stays full-width, matching existing responsive behavior.
- Editor content (Monaco `CodeEditor`, `MarkdownEditor`, markdown preview) reflows/relayouts to fill the resized width so more code/content is visible without horizontal scrolling.

## Notes

- Current drawer: `src/components/items/ItemDrawer.tsx` renders `<SheetContent side="right" className="w-full sm:w-120 sm:max-w-120 p-0 flex flex-col gap-0">` — fixed width, no resize affordance today.
- `SheetContent` (`src/components/ui/sheet.tsx`) is a shared Base UI Dialog wrapper; `ItemDrawer.tsx` is currently its only consumer (confirmed via grep), so resize logic can live in `ItemDrawer.tsx` (or a small hook) without touching the generic primitive.
- Monaco doesn't auto-reflow on parent container resize — `CodeEditor.tsx` will likely need an explicit `editor.layout()` call (or a `ResizeObserver`) triggered while dragging.
